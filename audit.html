Description

Serveur HTTP Python minimaliste permettant de télécharger des vidéos YouTube et TikTok en MP3, MP4 ou AVI. Cas d'usage principal : conversion pour le lecteur portable Aigo (format AVI H264 all-intra, 20fps, 240p portrait).

✅ Points forts
  • ✓ Single-file, zéro dépendance Python
  • ✓ UI web intégrée dans le script
  • ✓ Compatibilité Aigo validée et documentée
  • ✓ Auto-delete fichiers après download
  • ✓ Protection path traversal
  • ✓ Support YouTube + TikTok
  • ✓ Déployé sur serveur DE (systemd)
⚠️ Points d'attention
  • ⚠ Pas d'authentification (accès public sur DE)
  • ⚠ Traitement synchrone (1 download à la fois)
  • ⚠ Pas de queue, timeout client possible
  • ⚠ Fichiers temporaires si crash ffmpeg
  • ⚠ Limite 3min pour Aigo (contrainte hardware)
  • ⚠ TikTok: warning impersonation yt-dlp

Aigo Player — Reverse Engineering

Lecteur vidéo portable chinois avec décodeur H264 hardware limité. Nécessite un encodage très spécifique pour fonctionner correctement.

Contraintes découvertes (par tests exhaustifs)

ContrainteSymptômeSolution
Pas de P-frames supportées Glitches verts 50%, 1fps, freeze -g 1 (all-intra)
Limite ~3600 frames all-intra Coupure exacte à 2:01 à 30fps -r 20 → max 3:00
Déblocking filter Artefacts si activé no-deblock=1
SPS non standard Freeze au démarrage patch_avi_sps()
Affichage portrait (rotation) Vidéo à 90° sur l'écran transpose=2 dans vf
Résolution max Décodage trop lent 288×240 avant rotation

Tests GOP effectués

-g 30 — Glitches intermittents (player perd sync sans SPS fréquent)
-g 5 — Pire : 1fps + artefacts (P-frames incompatibles)
-g 2 — Très lent, glitches verts sur 50% de l'image
-g 1 — Parfait (all-intra) mais limite 2:01 à 30fps
-g 1 + -r 20 — ✅ SOLUTION FINALE : vidéo complète jusqu'à 3:00

Détail encodage Aigo

ffmpeg -y -i input \
  -c:v libx264 -profile:v baseline -level:v 4.1 \
  -preset ultrafast -tune fastdecode \
  -x264-params "bframes=0:ref=1:annexb=1:no-deblock=1:no-psy=1:no-mbtree=1:\
    aq-mode=0:chroma-qp-offset=0:partitions=none:me=dia:subme=0:\
    trellis=0:weightp=0:colorprim=undef:transfer=undef:colormatrix=undef" \
  -qp 38 -g 1 \
  -vtag H264 \
  -vf "scale=288:162,transpose=2,pad=240:288:(240-iw)/2:0:black,setsar=1:1" \
  -r 20 \
  -c:a pcm_s16le -ar 16000 -ac 1 \
  output.avi

patch_avi_sps()

Fonction de patch binaire qui :

  • Remplace le SPS H264 par une version minimale : 000000016742c029da0f0964
  • Supprime les SEI NAL units (remplacées par des fillers de même taille)
  • Opère en-place sur le fichier AVI (scan du contenu binaire)
  • Uniquement appelée pour le mode Aigo (is_aigo=True)

Encodage MP4 (mobile)

ffmpeg -y -i input \
  -c:v libx264 -profile:v high -level:v 4.1 \
  -preset fast -crf 22 \
  -c:a aac -b:a 128k -ar 44100 -ac 2 \
  -movflags +faststart \
  output.mp4

Compatible : QuickTime, iOS, Android, navigateurs. +faststart permet la lecture en streaming progressif.

Historique des bugs résolus

🐛 Bug #1 — Coupure à 2:01 sur l'Aigo Résolu

Symptôme : Toutes les vidéos > 2:01 se coupaient exactement à 2:01 puis passaient à la suivante.

Diagnostic : Limite hardware Aigo ~3600 frames pour H264 all-intra. À 30fps : 173s × 30 = 5190 frames > 3600.

Fix : -r 20 → 173s × 20 = 3460 frames < 3600 → vidéo complète.

🐛 Bug #2 — Glitches verts + 1fps avec P-frames Résolu

Symptôme : En essayant -g 2, -g 5, -g 30 pour corriger le bug #1 : image intermittente, glitches verts sur 50% de l'écran, playback à 1fps.

Diagnostic : Le décodeur hardware Aigo ne supporte pas les P-frames avec notre encodage x264 (no-deblock interférait avec le décodeur hardware).

Fix : Retour à -g 1 (all-intra) + -r 20 pour contourner la limite frames.

🐛 Bug #3 — MP4 non compatible QuickTime Résolu

Symptôme : Les MP4 YouTube téléchargés s'ouvraient dans VLC mais pas QuickTime Player.

Diagnostic : yt-dlp sélectionnait AV1+Opus (YouTube Shorts). QuickTime ne supporte pas AV1.

Fix : Transcode systématique via ffmpeg en H264+AAC avec -movflags +faststart.

🐛 Bug #4 — TikTok : Requested format not available Résolu

Symptôme : Les liens TikTok échouaient avec "Requested format is not available".

Diagnostic : TikTok fournit des formats combinés (video+audio ensemble), pas de flux séparés. Le format bestvideo+bestaudio ne fonctionnait pas.

Fix : Format bestvideo[height<=720]+bestaudio/best[height<=720]/best avec fallback.
